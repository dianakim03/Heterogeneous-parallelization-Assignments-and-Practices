# Лабораторная работа №3  
## Тема: Реализация более сложных алгоритмов сортировки (слияние, быстрая, пирамидальная) на GPU с использованием CUDA



## Цель работы
- Освоить программирование на CUDA и основы параллельных вычислений на GPU.
- Реализовать и оптимизировать сложные алгоритмы сортировки на GPU (слияние, быстрая сортировка, пирамидальная).
- Исследовать производительность алгоритмов на GPU и сравнить их с реализациями на CPU.



## Описание работы
В рамках лабораторной работы реализованы следующие алгоритмы сортировки:

- Merge Sort
- Quick Sort
- Heap Sort

Для каждого алгоритма:
- реализована последовательная версия на CPU;
- реализована параллельная версия на GPU с использованием CUDA;
- выполнено измерение времени выполнения;
- проведено сравнение производительности CPU и GPU.



## Структура
Все задания лабораторной работы объединены в один файл `main.cu`, потому что:

- все алгоритмы работают с одними и теми же входными данными;
- используется единый механизм измерения времени;
- удобнее сравнивать CPU и GPU реализации в одном месте;
- код становится логически целостным и понятным.

## Реализация заданий
### Задание 1. Параллельная сортировка слиянием на CUDA
Массив делится на подмассивы фиксированного размера (CHUNK).
Каждый подмассив обрабатывается отдельным CUDA-блоком и сортируется параллельно.
После этого выполняется поэтапное слияние отсортированных подмассивов по парам до получения полностью отсортированного массива.

### Задание 2. Параллельная быстрая сортировка на CUDA
Параллельность достигается за счёт того, что каждый CUDA-блок сортирует свою часть массива.
Внутри блока используется упрощённая сортировка (insertion sort), что делает реализацию стабильной и наглядной для учебных целей.

### Задание 3. Параллельная пирамидальная сортировка на CUDA
Для каждого подмассива, обрабатываемого CUDA-блоком, строится бинарная куча.
Извлечение элементов и восстановление свойств кучи выполняется внутри блока.
Параллельность достигается за счёт одновременной обработки нескольких подмассивов.

### Задание 4. Сравнение производительности CPU и GPU
Для всех алгоритмов реализованы последовательные версии на CPU.
Время выполнения измеряется:
на CPU — с использованием std::chrono;
на GPU — с использованием cudaEvent.

Эксперименты выполняются для массивов следующих размеров:
10 000 элементов;
100 000 элементов;
1 000 000 элементов.


## Общая логика работы программы

Генерируется массив случайных целых чисел заданного размера.
Один и тот же массив используется для сортировки на CPU и GPU.
Для каждого алгоритма измеряется время выполнения.
После сортировки выполняется проверка корректности результата.
Результаты выводятся в консоль для анализа.
