# Часть 1: Параллельный стек LIFO на GPU (CUDA)

В первой части лабораторной работы реализован параллельный стек (LIFO) на GPU с использованием **глобальной памяти** и **атомарных операций**.  
Цель работы — показать, как несколько потоков могут безопасно выполнять операции добавления и извлечения элементов без конфликтов данных.

## Что реализовано в данной части

1) Инициализация стека
- Создаётся стек фиксированной ёмкости `CAP`.
- Для хранения элементов используется массив в глобальной памяти GPU.
- Переменная `top` задаёт вершину стека и инициализируется как пустая.

2) Параллельная операция push
- Несколько GPU-потоков одновременно выполняют операцию `push`.
- Каждый поток пытается добавить своё значение (идентификатор потока) в стек.
- Обновление вершины стека выполняется с помощью атомарных операций, что предотвращает конфликты.

3) Параллельная операция pop
- Несколько GPU-потоков одновременно выполняют операцию `pop`.
- Каждый поток извлекает один элемент из стека в выходной массив.
- Атомарные операции гарантируют, что один элемент не будет извлечён несколькими потоками.

4) Проверка результата на CPU
После выполнения GPU-ядер результаты копируются на CPU и выводятся:
- количество успешных операций `push`
- количество успешных операций `pop`
- финальное значение вершины стека `top`
- первые 10 извлечённых элементов
