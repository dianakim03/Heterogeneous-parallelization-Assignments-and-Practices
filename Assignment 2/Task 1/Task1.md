# Задача 1. Введение в гетерогенную параллелизацию

Гетерогенная параллелизация — это подход к параллельным вычислениям, при котором одна программа использует несколько разных типов вычислительных устройств, чаще всего центральный процессор CPU и графический процессор GPU. Основная идея заключается в том, чтобы распределить работу между устройствами таким образом, чтобы каждое из них выполняло те задачи, для которых оно подходит лучше всего. CPU обычно отвечает за управление программой, сложную логику и подготовку данных, а GPU используется для выполнения большого количества однотипных вычислений над массивами данных.

Параллельные вычисления на CPU и GPU существенно различаются по своей архитектуре и принципу работы. CPU имеет небольшое количество мощных ядер, которые хорошо справляются с последовательным кодом, ветвлениями, условиями и сложными структурами данных. GPU, в свою очередь, содержит тысячи простых вычислительных потоков и наиболее эффективен при выполнении одинаковых операций над большим числом элементов. Поэтому GPU показывает высокую производительность в задачах, где можно выполнять одну и ту же инструкцию параллельно, тогда как CPU лучше подходит для задач с нерегулярной логикой и частыми переходами.

Основным преимуществом гетерогенной параллелизации является повышение производительности за счёт более наилучшего использования аппаратных ресурсов. При таком подходе CPU и GPU не простаивают, а работают совместно, каждый над своей частью задачи. Это позволяет значительно ускорить выполнение программ при больших объёмах данных. Кроме того, гетерогенная параллелизация даёт возможность улучшить масштабируемость приложений и, в ряде случаев, снизить энергопотребление за счёт переноса тяжёлых вычислений на GPU.

Гетерогенная параллелизация широко применяется в реальных приложениях. В компьютерной графике и видеоиграх GPU отвечает за рендеринг изображений и обработку графических эффектов, а CPU управляет логикой игры и поведением объектов. В машинном обучении и нейронных сетях GPU используется для выполнения основных вычислений, таких как умножение матриц и обучение моделей, в то время как CPU занимается подготовкой данных и управлением процессом обучения. В задачах обработки изображений и видео GPU ускоряет фильтрацию и анализ кадров, а CPU координирует работу программы. Также гетерогенная параллелизация активно используется в научных расчётах, моделировании физических процессов и инженерных задачах.
