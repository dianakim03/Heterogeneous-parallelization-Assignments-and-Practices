# Task 2. Влияние размера блока потоков на производительность (CUDA)

## Цель
Исследовать влияние размера блока потоков на производительность CUDA-программы при выполнении поэлементного сложения двух массивов.

## Что делается в программе

Программа запускается несколько раз с разными размерами блока потоков, после чего сравнивается время выполнения CUDA-ядра для каждого варианта.

1) Подготовка данных
- Создаются два массива `a` и `b` размером `N = 1 000 000` на CPU.
- Массивы инициализируются простыми значениями для удобной проверки корректности.

2) Копирование данных на GPU
- Выделяется память в глобальной памяти GPU.
- Массивы `a` и `b` копируются с CPU на GPU.

3) Запуск CUDA-ядра
- Выполняется поэлементное сложение массивов на GPU.
- Ядро запускается несколько раз с разными размерами блока потоков (например, 128, 256 и 512).

4) Замер времени
- Для каждого размера блока измеряется время выполнения ядра с использованием `cudaEvent`.

5) Проверка корректности
- Результат копируется обратно на CPU.
- Проверяется корректность выполнения операции сложения.


## Ожидаемый результат
- Для разных размеров блока будут получены разные времена выполнения.
- Как правило, оптимальный размер блока обеспечивает наименьшее время выполнения.
- Результаты вычислений остаются корректными для всех вариантов запуска.
