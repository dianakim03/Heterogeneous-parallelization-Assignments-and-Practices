# Task 4 — Подбор оптимальной конфигурации grid/block (CUDA)

## Цель
Подобрать более оптимальную конфигурацию запуска CUDA-ядра (размер блока и количество блоков) и сравнить производительность:
- неоптимальной конфигурации
- оптимальной конфигурации

В качестве основы используется одна из программ из предыдущих заданий.


## Что делается в задании
1. Выбирается одно CUDA-ядро, которое выполняет простую поэлементную операцию над массивом размера `N = 1 000 000`.
2. Программа запускается минимум в двух вариантах:
   - неоптимальный вариант (маленький block size, который хуже загружает GPU)
   - оптимальный вариант (block size, который показывает наименьшее время на данном GPU)
3. Для каждого варианта измеряется время выполнения с помощью `cudaEvent`.
4. Выполняется проверка корректности результата.
5. Делается краткий вывод: какая конфигурация быстрее и почему.


## Идея оптимизации 
Размер блока влияет на то, насколько эффективно GPU распределяет потоки по своим вычислительным ресурсам.  
Если блок слишком маленький, GPU может быть загружен не полностью, и время будет больше.  
При более подходящем размере блока (например, 256 или 512) обычно достигается лучшее время выполнения.

## Ожидаемый результат
- обе конфигурации должны выдавать корректные результаты (`ok`)
- оптимальная конфигурация должна показывать меньшее время выполнения
